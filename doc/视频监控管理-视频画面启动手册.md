# 视频监控管理 - 视频画面启动手册

## 1. 适用范围

本手册用于当前仓库本地联调环境，目标是让 `video-monitor` 页面真正显示 YOLO 识别视频流，而不是只打开前端页面。

当前项目的视频识别链路分两部分：

1. 主项目 `yingzi-zncf`
2. 外部 YOLO 联调项目 `/Users/guanyiru/Desktop/yingzicode/摄像头识别`

---

## 2. 当前真实接入方式

当前项目并没有把 YOLO 推理进程直接内嵌到主项目里。

视频监控管理页面的数据来源是：

1. 主项目 `device-service` 返回监控列表和 `analysisStreamUrl`
2. `analysisStreamUrl` 默认指向本机 YOLO 联调服务

当前默认口径：

- 识别流基础地址：`http://127.0.0.1:18081`
- 标注流路径：`/annotated.mjpg`
- 原始流路径：`/raw.mjpg`

对应配置文件：

- [backend/device-service/src/main/resources/application.yml](/Users/guanyiru/Desktop/yingzicode/yingzi-zncf/backend/device-service/src/main/resources/application.yml)
- [scripts/dev/common.sh](/Users/guanyiru/Desktop/yingzicode/yingzi-zncf/scripts/dev/common.sh)

---

## 3. 启动步骤

## 3.1 启动主项目

在项目根目录执行：

```bash
cd /Users/guanyiru/Desktop/yingzicode/yingzi-zncf
./scripts/dev/start-all.sh --takeover
```

验证：

```bash
./scripts/dev/status-all.sh
```

预期看到：

- `gateway-service` `managed-up`
- `auth-service` `managed-up`
- `device-service` `managed-up`
- `frontend` `managed-up`

---

## 3.2 启动 YOLO 联调服务

在新终端执行：

```bash
cd /Users/guanyiru/Desktop/yingzicode/摄像头识别
source .venv/bin/activate
python review_app.py --config config.yaml --host 127.0.0.1 --port 18081
```

启动后不要关闭这个终端。

如果你希望后台运行，可自行使用 `nohup` 或 `tmux`，但本手册默认采用前台保活，最容易观察错误日志。

---

## 3.3 验证识别流

在第三个终端执行：

```bash
curl -I 'http://127.0.0.1:18081/annotated.mjpg'
```

正确结果应包含：

```text
HTTP/1.1 200 OK
Content-Type: multipart/x-mixed-replace; boundary=frame
```

如需验证原始流：

```bash
curl -I 'http://127.0.0.1:18081/raw.mjpg'
```

---

## 3.4 打开前端页面

登录：

- `http://localhost:5175/login`
- 用户名：`admin`
- 密码：`admin`

进入：

- `http://localhost:5175/video-monitor`

如果页面之前已经打开过，且显示的是旧失败态，做一次硬刷新。

## 4. 视频源切换

YOLO 联调项目实际拉取的视频源由下面文件控制：

- [/Users/guanyiru/Desktop/yingzicode/摄像头识别/config.yaml](/Users/guanyiru/Desktop/yingzicode/摄像头识别/config.yaml)

## 5. 常见问题排查

## 5.1 页面提示“视频加载失败”

按下面顺序检查：

1. 主项目是否启动：

```bash
./scripts/dev/status-all.sh
```

2. `device-service` 是否正常：

```bash
lsof -iTCP:8088 -sTCP:LISTEN -n -P
```

3. YOLO 联调服务是否正常：

```bash
lsof -iTCP:18081 -sTCP:LISTEN -n -P
```

4. 标注流是否可访问：

```bash
curl -I 'http://127.0.0.1:18081/annotated.mjpg'
```

5. RTSP 源是否可连：
- 查看 `review_app.py` 启动终端日志
- 若摄像头离线、账号密码错误、网络不通，YOLO 服务虽然启动了，但不一定能持续产出画面

---

## 5.2 `18081` 端口被占用

先查占用：

```bash
lsof -iTCP:18081 -sTCP:LISTEN -n -P
```

如果发现占用进程不是 `review_app.py`，需要先释放端口，再启动 YOLO 联调服务。

当前项目口径固定使用 `18081`，原因是：

1. `8081` 已经用于 `auth-service`
2. 视频识别流不能再和登录端口复用

---

## 5.3 主项目启动了，但监控列表为空

这通常是以下原因之一：

1. 当前登录用户无对应组织数据权限
2. `device_info` 中没有可见的摄像头数据
3. 摄像头没有在 `device-service` 返回列表中

先检查设备管理页是否能看到摄像头记录：

- `http://localhost:5175/device`

---

## 5.4 我改了端口，页面还是不出画

如果你没有使用 `18081`，那主项目也必须同步改：

- `VISION_STREAM_BASE_URL`

当前默认值定义在：

- [scripts/dev/common.sh](/Users/guanyiru/Desktop/yingzicode/yingzi-zncf/scripts/dev/common.sh)

改完后至少要重启：

1. `device-service`
2. 前端页面刷新

---

## 6. 推荐联调方式

推荐固定采用下面顺序：

1. 启动主项目
2. 启动 `review_app.py`
3. 用 `curl` 验证 `annotated.mjpg`
4. 登录后打开 `video-monitor`

不要先开页面再猜测为什么没图，这样最容易把问题混在一起。
