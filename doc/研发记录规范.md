# 研发记录规范

## 目的

用于沉淀本项目每日研发操作、需求改动、实现范围、验证结果和后续待办，保证开发过程可追踪、可复盘、可交接。

## 记录位置

- 每日研发日志统一记录在 `doc/研发记录/`
- 建议按月份维护文件，例如：`doc/研发记录/2026-06.md`

## 记录原则

- 每个自然日新增一个日期章节
- 同一天内可追加多个时间段记录
- 只记录真实发生的研发动作，不写空泛总结
- 每条记录至少包含：背景、改动、验证、待办

## 推荐模板

```md
## YYYY-MM-DD

### HH:mm 事项标题

#### 背景
- 为什么做

#### 本次改动
- 改了哪些页面、模块、接口或文档

#### 验证
- 构建、测试、联调或手工验证结果

#### 后续
- 剩余问题、下一步计划、风险点
```

## 当前执行约定

- 从 2026-06-01 起，dashboard 与相关联动开发统一按本规范记录
- 后续新增开发动作时，优先追加到当月研发日志文件中
