# 第一部分：数据库选型与适配说明

## 1.1 MySQL 8.0 选型理由

### 1.1.1 技术兼容性

**与Spring Boot 3.2完美适配**
- Spring Boot 3.2官方推荐MySQL 8.0+作为生产环境数据库
- 原生支持MySQL 8.0新特性（窗口函数、CTE、JSON增强）
- JDBC驱动完全兼容（mysql-connector-java 8.0.33+）

**与MyBatis Plus深度集成**
- MyBatis Plus 3.5.x完全支持MySQL 8.0
- 自动适配MySQL 8.0的主键策略（AUTO_INCREMENT）
- 分页插件原生支持MySQL 8.0的LIMIT语法
- 条件构造器完美映射MySQL 8.0的SQL特性

### 1.1.2 微服务共享数据库特性

**多服务共享单库架构**
- 采用表名前缀隔离策略（如`scm_`、`wms_`、`auth_`）
- 通过Nacos统一管理数据库连接配置
- 支持服务间通过HTTP调用共享数据访问
- 避免分布式事务复杂性，保证数据一致性

**性能与资源优化**
- 单库架构减少连接池开销（适配JVM Xms256m-Xmx512m）
- 减少跨库JOIN查询，提升查询性能
- 简化运维部署，降低数据库实例成本

### 1.1.3 MySQL 8.0核心优势

| 特性 | 优势 | 应用场景 |
|------|------|---------|
| InnoDB增强 | 更高并发性能、更快的DDL操作 | 高并发采购订单、库存更新 |
| JSON支持 | 原生JSON字段类型、JSON函数 | 设备配置、告警详情、审计日志 |
| 窗口函数 | 复杂统计分析 | 库存趋势、营养分析、成本统计 |
| CTE（公用表表达式） | 递归查询组织树 | 多级组织架构查询 |
| 角色与权限 | 细粒度权限控制 | 数据库层面权限隔离 |
| 默认UTF8MB4 | 完整Unicode支持（含Emoji） | 用户评价、反馈内容 |

## 1.2 Nacos配置联动

### 1.2.1 数据库连接配置（Nacos配置中心）

**配置文件：** `application-mysql.yml`（存储在Nacos配置中心）

```yaml
# Nacos配置项：kitchen-platform-mysql.yml
# Data ID: kitchen-platform-mysql.yml
# Group: DEFAULT_GROUP
# Nacos Server: 192.168.31.4:8848

spring:
  datasource:
    # 主数据源配置
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://${MYSQL_HOST:192.168.31.4}:${MYSQL_PORT:3306}/kitchen_platform?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true&rewriteBatchedStatements=true
    username: ${MYSQL_USERNAME:kitchen_admin}
    password: ${MYSQL_PASSWORD:Kitchen@2026}

    # HikariCP连接池配置（适配JVM Xms256m-Xmx512m）
    hikari:
      minimum-idle: 5                    # 最小空闲连接数
      maximum-pool-size: 20              # 最大连接池大小（适配小内存）
      connection-timeout: 30000          # 连接超时时间（30秒）
      idle-timeout: 600000               # 空闲连接超时时间（10分钟）
      max-lifetime: 1800000              # 连接最大生命周期（30分钟）
      connection-test-query: SELECT 1    # 连接测试查询（健康检查）
      pool-name: KitchenHikariCP

# MyBatis Plus配置
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true   # 下划线转驼峰
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl  # SQL日志
  global-config:
    db-config:
      id-type: AUTO                      # 主键策略：数据库自增
      logic-delete-field: deleted        # 逻辑删除字段
      logic-delete-value: 1              # 逻辑删除值
      logic-not-delete-value: 0          # 逻辑未删除值
```

## 1.3 数据库命名规范

### 1.3.1 表命名规范

**格式：** `{服务前缀}_{业务模块}_{表名}`

| 服务前缀 | 说明 | 示例 |
|---------|------|------|
| auth_ | 认证授权服务 | auth_user, auth_role |
| scm_ | 供应链管理服务 | scm_supplier, scm_purchase_order |
| wms_ | 仓储管理服务 | wms_warehouse, wms_inventory |
| recipe_ | 菜谱营养服务 | recipe_menu, recipe_plan |
| cook_ | 烹饪记录服务 | cook_task, cook_record |
| sample_ | 留样管理服务 | sample_record, sample_disposal |
| health_ | 健康晨检服务 | health_check, health_certificate |
| device_ | 设备管理服务 | device_info, device_alert |
| sys_ | 系统基础服务 | sys_org, sys_dict |

### 1.3.2 字段命名规范

- **主键：** `id`（BIGINT自增）
- **外键：** `{关联表名}_id`（如`user_id`、`supplier_id`）
- **状态字段：** `status`（VARCHAR，存储枚举值）
- **时间字段：** `created_at`、`updated_at`、`deleted_at`（DATETIME）
- **逻辑删除：** `deleted`（TINYINT，0=未删除，1=已删除）
- **租户隔离：** `tenant_id`（BIGINT，多租户场景）
- **组织隔离：** `org_id`（BIGINT，组织数据隔离）

### 1.3.3 索引命名规范

- **主键索引：** `PRIMARY`（自动创建）
- **唯一索引：** `uk_{表名}_{字段名}`（如`uk_user_email`）
- **普通索引：** `idx_{表名}_{字段名}`（如`idx_order_status`）
- **联合索引：** `idx_{表名}_{字段1}_{字段2}`（如`idx_order_user_status`）

# 第二部分：核心表结构设计

## 2.1 认证授权模块（auth_）

### 2.1.1 用户表（auth_user）

**表说明：** 存储系统用户基础信息，适配JWT认证

```sql
CREATE TABLE `auth_user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',
  `password` VARCHAR(255) NOT NULL COMMENT '密码（BCrypt加密）',
  `email` VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
  `phone` VARCHAR(20) DEFAULT NULL COMMENT '手机号',
  `real_name` VARCHAR(50) DEFAULT NULL COMMENT '真实姓名',
  `avatar_url` VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
  `gender` TINYINT DEFAULT 0 COMMENT '性别：0=未知，1=男，2=女',
  `status` VARCHAR(20) NOT NULL DEFAULT 'active' COMMENT '状态：active=启用，inactive=禁用，locked=锁定',
  `last_login_at` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `last_login_ip` VARCHAR(50) DEFAULT NULL COMMENT '最后登录IP',
  `login_fail_count` INT DEFAULT 0 COMMENT '登录失败次数',
  `locked_until` DATETIME DEFAULT NULL COMMENT '锁定截止时间',
  `org_id` BIGINT NOT NULL COMMENT '所属组织ID',
  `tenant_id` BIGINT NOT NULL DEFAULT 1 COMMENT '租户ID（多租户隔离）',
  `created_by` BIGINT DEFAULT NULL COMMENT '创建人ID',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_by` BIGINT DEFAULT NULL COMMENT '更新人ID',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除：0=未删除，1=已删除',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_user_username` (`username`, `deleted`),
  UNIQUE KEY `uk_user_email` (`email`, `deleted`),
  UNIQUE KEY `uk_user_phone` (`phone`, `deleted`),
  KEY `idx_user_org` (`org_id`),
  KEY `idx_user_tenant` (`tenant_id`),
  KEY `idx_user_status` (`status`),
  KEY `idx_user_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

**MyBatis Plus实体类示例：**

```java
@Data
@TableName("auth_user")
public class User implements Serializable {

    @TableId(value = "id", type = IdType.AUTO)
    private Long id;

    private String username;

    private String password;

    private String email;

    private String phone;

    @TableField("real_name")
    private String realName;

    @TableField("avatar_url")
    private String avatarUrl;

    private Integer gender;

    private String status;

    @TableField("last_login_at")
    private LocalDateTime lastLoginAt;

    @TableField("last_login_ip")
    private String lastLoginIp;

    @TableField("login_fail_count")
    private Integer loginFailCount;

    @TableField("locked_until")
    private LocalDateTime lockedUntil;

    @TableField("org_id")
    private Long orgId;

    @TableField("tenant_id")
    private Long tenantId;

    @TableField(value = "created_by", fill = FieldFill.INSERT)
    private Long createdBy;

    @TableField(value = "created_at", fill = FieldFill.INSERT)
    private LocalDateTime createdAt;

    @TableField(value = "updated_by", fill = FieldFill.UPDATE)
    private Long updatedBy;

    @TableField(value = "updated_at", fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updatedAt;

    @TableLogic
    private Integer deleted;
}
```

### 2.1.2 JWT令牌表（auth_token）

```sql
CREATE TABLE `auth_token` (

  `access_token` VARCHAR(500) NOT NULL COMMENT 'JWT访问令牌',

  `refresh_token_expires_at` DATETIME NOT NULL COMMENT '刷新令牌过期时间',
  `device_type` VARCHAR(20) DEFAULT NULL COMMENT '设备类型：web/mobile/tablet',
  `device_id` VARCHAR(100) DEFAULT NULL COMMENT '设备唯一标识',
  `ip_address` VARCHAR(50) DEFAULT NULL COMMENT '登录IP地址',
  `user_agent` VARCHAR(500) DEFAULT NULL COMMENT '用户代理信息',
  `is_revoked` TINYINT NOT NULL DEFAULT 0 COMMENT '是否已撤销：0=否，1=是',
  `revoked_at` DATETIME DEFAULT NULL COMMENT '撤销时间',
  `revoked_reason` VARCHAR(200) DEFAULT NULL COMMENT '撤销原因',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_token_access` (`access_token`(255)),
  UNIQUE KEY `uk_token_refresh` (`refresh_token`(255)),
  KEY `idx_token_user` (`user_id`),
  KEY `idx_token_expires` (`access_token_expires_at`),
  KEY `idx_token_revoked` (`is_revoked`)
```

## 2.10 表关联关系说明（ER + 微服务适配）

> ✅ 微服务适配补充：本章节基于现有表结构进行关系补全，不改变原有字段与建表语句。
> ✅ 微服务适配补充：关系划分为「同服务内关联」与「跨服务关联」，跨服务默认采用“业务外键（逻辑外键）+ API/事件同步”模式。
> ✅ 微服务适配补充：在共享 MySQL 架构下，**仅同服务核心强一致链路建议使用物理外键**；跨服务及高并发表建议使用逻辑外键，避免锁放大与DDL耦合。

### 2.10.1 关系总原则（适配 Spring Cloud 2023）

- ✅ 微服务适配补充：**同服务内关联**
  - 优先可落地物理外键（InnoDB），并补充联合索引，保障 MP 分页/条件查询性能。
- ✅ 微服务适配补充：**跨服务关联**
  - 使用逻辑外键（仅字段约定+索引），不强制 DB 外键；
  - 读取侧通过 Spring Cloud Gateway 转发服务 API 聚合；
  - 写入侧通过领域事件（Outbox/MQ）或定时对账任务保证最终一致。
- ✅ 微服务适配补充：**多对多关系**
  - 必须通过中间表实现（已存在：`auth_user_role`、`auth_role_permission`、`recipe_menu_ingredient`、`recipe_plan_item`）；
  - 中间表建议统一审计字段（`created_at`、`updated_at`、`created_by`）以适配 MyBatis Plus 通用 Mapper。

### 2.10.7 跨服务数据同步策略（Nacos + Gateway）

- ✅ 微服务适配补充：**查询同步（读时聚合）**
  - 通过 Gateway 统一入口：如 Cook 服务查询 `recipe_menu`、`recipe_plan` 基础信息；
  - 结果短缓存（如 1~5 分钟）降低跨服务调用频率。
- ✅ 微服务适配补充：**事件同步（写后异步）**
  - 关键主数据（组织、员工、菜谱、物料）变更发布领域事件；
  - 消费端刷新冗余字段（`*_name`、状态快照）与本地缓存。
- ✅ 微服务适配补充：**定时对账（兜底）**
  - 按 Nacos 配置任务频率（如每 30/60 分钟）执行主键一致性巡检；
  - 输出差异清单并自动修复可回补字段。
- ✅ 微服务适配补充：**失败补偿**
  - 采用“重试 + 死信队列 + 人工补偿台账”；
  - 保证最终一致，避免分布式强事务。

# 第三部分：索引与性能优化设计

## 3.1 索引设计原则

### 3.1.1 MyBatis Plus分页查询优化

**分页查询特点：**
- MyBatis Plus分页插件自动添加`LIMIT offset, size`
- 需要在WHERE条件字段上建立索引
- 避免深分页（offset过大导致性能下降）

**索引策略：**
```sql
-- 示例：采购订单分页查询
-- 查询条件：status + org_id + created_at
-- 索引设计：联合索引覆盖查询条件
CREATE INDEX idx_order_status_org_created 
ON scm_purchase_order(status, org_id, created_at DESC);
```

### 3.1.2 条件构造器查询优化

**MyBatis Plus条件构造器常见查询：**
```java
// 示例：多条件查询
QueryWrapper<Supplier> wrapper = new QueryWrapper<>();
wrapper.eq("status", "active")
       .eq("org_id", orgId)
       .like("supplier_name", keyword)
       .orderByDesc("created_at");
```

**对应索引设计：**
```sql
-- 精确匹配字段优先，模糊查询字段靠后
CREATE INDEX idx_supplier_status_org 
ON scm_supplier(status, org_id);

-- 全文搜索字段单独索引
CREATE FULLTEXT INDEX ft_supplier_name 
ON scm_supplier(supplier_name);
```

## 3.3 JVM内存优化建议（Xms256m-Xmx512m）

### 3.3.1 连接池配置优化

**HikariCP连接池参数（已在Nacos配置）：**
```yaml
hikari:
  minimum-idle: 5              # 最小空闲连接（减少内存占用）
  maximum-pool-size: 20        # 最大连接数（适配小内存）
  connection-timeout: 30000    # 连接超时30秒
  idle-timeout: 600000         # 空闲连接10分钟回收
  max-lifetime: 1800000        # 连接最大生命周期30分钟
```

**内存占用估算：**
- 单个MySQL连接约占用2-4MB内存
- 20个连接池最大占用：20 × 4MB = 80MB
- 预留内存：512MB - 80MB = 432MB（应用可用）

### 3.3.2 索引内存优化

**索引选择性原则：**
- 高选择性字段优先建索引（如唯一字段、状态字段）
- 避免在低选择性字段建索引（如性别、是否删除）
- 联合索引遵循最左前缀原则

**索引大小控制：**
```sql
-- 查询索引大小
SELECT 
  TABLE_NAME,
  INDEX_NAME,
  ROUND(STAT_VALUE * @@innodb_page_size / 1024 / 1024, 2) AS size_mb
FROM mysql.innodb_index_stats
WHERE DATABASE_NAME = 'kitchen_platform'
  AND STAT_NAME = 'size'
ORDER BY STAT_VALUE DESC;
```

### 3.3.3 查询优化建议

**避免全表扫描：**
```sql
-- ❌ 错误：未使用索引
SELECT * FROM scm_purchase_order WHERE DATE(created_at) = '2026-03-16';

-- ✅ 正确：使用索引范围查询
SELECT * FROM scm_purchase_order 
WHERE created_at >= '2026-03-16 00:00:00' 
  AND created_at < '2026-03-17 00:00:00';
```

**分页查询优化：**
```sql
-- ❌ 错误：深分页性能差
SELECT * FROM scm_purchase_order 
ORDER BY created_at DESC 
LIMIT 10000, 20;

-- ✅ 正确：使用游标分页
SELECT * FROM scm_purchase_order 
WHERE id < last_id 
ORDER BY id DESC 
LIMIT 20;
```

### 4.3.2 应用层健康检查

```java
// Spring Boot Actuator健康检查
@Component
public class DatabaseHealthIndicator implements HealthIndicator {
    
    @Autowired
    private DataSource dataSource;
    
    @Override
    public Health health() {
        try (Connection conn = dataSource.getConnection()) {
            Statement stmt = conn.createStatement();
            ResultSet rs = stmt.executeQuery("SELECT 1");
            if (rs.next()) {
                return Health.up()
                    .withDetail("database", "MySQL 8.0")
                    .withDetail("status", "UP")
                    .build();
            }
        } catch (Exception e) {
            return Health.down()
                .withDetail("error", e.getMessage())
                .build();
        }
        return Health.down().build();
    }
}
```
