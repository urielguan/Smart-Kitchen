## 📄 各文件说明

### 1. index.html（主入口）
- **作用**：HTML 结构骨架，引入所有 CSS 和 JS 文件
- **关键点**：
  - 引入 Element Plus CSS（CDN）
  - 按顺序加载 7 个 JS 文件（顺序不可颠倒）
  - 包含全局组件容器（Toast/弹窗/AI提示框）

### 2. css/main.css（全局样式）
- **作用**：所有页面样式（布局/组件/动画）
- **包含模块**：
  - 全局重置 & 布局容器
  - 侧边栏样式
  - 顶部导航栏
  - 统计卡片
  - 工具栏 & 按钮
  - 数据表格
  - 状态标签（Tag）
  - 弹窗 & 遮罩层
  - 表单组件
  - Toast 提示
  - 标签页（Tabs）
  - AI 提示框
  - 占位页面

### 3. js/mock-data.js（模拟数据）
- **作用**：存储所有 Mock 数据
- **数据结构**：
  - `mockData.warehouses`：仓库列表（4条）
  - `mockData.inboundOrders`：入库单（3条）
  - `mockData.outboundOrders`：出库单（3条）
  - `mockData.stocktakeOrders`：盘点单（3条）
  - `mockData.materials`：物料列表（8条）
  - `mockData.recipes`：菜谱列表（6条）

### 4. js/utils.js（工具函数）
- **作用**：全局状态变量 + 公共工具函数
- **包含函数**：
  - `showToast(message, type)`：全局提示
  - `getStatusTag(status)`：状态标签生成
  - `showModal(content, width)`：打开弹窗
  - `closeModal()`：关闭弹窗
  - `showAiTooltip(text, event)`：AI 悬浮提示

### 5. js/sidebar.js（侧边栏导航）
- **作用**：侧边栏菜单配置 + 页面切换逻辑
- **包含函数**：
  - `menuConfig`：菜单结构配置（6个分组，12个菜单项）
  - `renderSidebar()`：渲染侧边栏高亮状态
  - `switchPage(page, group, label)`：切换页面
  - `renderPage(page)`：分发到对应模块渲染函数

### 6. js/warehouse.js（仓库管理模块）
- **作用**：仓库信息管理的完整交互逻辑
- **包含功能**：
  - 仓库列表：搜索/新增/编辑/删除
  - 入库单：搜索/新增/审核/删除
  - 出库单：搜索/审核/删除
  - 盘点单：搜索/审核
- **核心函数**：
  - `renderWarehousePage()`：主页入口
  - `switchWarehouseTab(tab)`：切换子标签页
  - `renderWarehouseList()`：仓库列表渲染
  - `renderInboundOrders()`：入库单渲染
  - `renderOutboundOrders()`：出库单渲染
  - `renderStocktakeOrders()`：盘点单渲染
  - `openWarehouseModal(id)`：新增/编辑仓库弹窗
  - `openInboundModal()`：新增入库单弹窗
  - `openAuditModal(id, type)`：审核弹窗

### 7. js/material.js（物料管理模块）
- **作用**：物料信息管理的完整交互逻辑
- **包含功能**：
  - 物料列表：搜索/新增/编辑/删除
  - AI 建议：自动填入临期提醒天数和预警期
- **核心函数**：
  - `renderMaterialPage()`：主页入口
  - `renderMaterialList()`：物料列表渲染
  - `searchMaterials()`：前端搜索过滤
  - `openMaterialModal(id)`：新增/编辑物料弹窗
  - `aiSuggestMaterial(event)`：AI 建议按钮

### 8. js/recipe.js（菜谱管理模块）
- **作用**：菜谱库管理的完整交互逻辑
- **包含功能**：
  - 菜谱列表：搜索/新增/编辑/删除
  - AI 建议：自动填入烹饪时长和目标温度
  - 食材列表：动态增删行
- **核心函数**：
  - `renderRecipePage()`：主页入口
  - `renderRecipeList()`：菜谱列表渲染
  - `searchRecipes()`：前端搜索过滤
  - `openRecipeModal(id)`：新增/编辑菜谱弹窗
  - `aiSuggestCookTime(event)`：AI 建议烹饪时长
  - `aiSuggestTemperature(event)`：AI 建议目标温度
  - `addIngredientRow()`：添加食材行
  - `removeIngredientRow(index)`：删除食材行

### 9. js/app.js（应用初始化）
- **作用**：页面加载完成后的初始化入口
- **包含函数**：
  - `init()`：初始化侧边栏、渲染默认页面、绑定全局事件

### 10. README.md（项目文档）
- **作用**：项目说明文档
- **包含内容**：
  - 快速开始指南
  - 项目结构说明

  - 技术栈介绍

## 📝 注意事项

1. **JS 加载顺序**：`index.html` 中的 JS 文件加载顺序不可颠倒，必须按以下顺序：
   ```
   mock-data.js → utils.js → sidebar.js → warehouse.js → material.js → recipe.js → app.js
   ```

2. **外部依赖**：需要联网加载 Element Plus CSS（CDN），如需离线使用请下载到本地

3. **数据持久化**：当前数据存储在内存中，刷新页面后恢复初始值，如需持久化可扩展 `localStorage`

4. **浏览器兼容**：不支持 IE 浏览器，推荐使用 Chrome 90+ / Firefox 88+ / Edge 90+
